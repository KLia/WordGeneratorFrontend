How it works

The Word Generator is built using a model called Markov Chains - a piece of software that utilizes statistics to determine what letters might occur next to each. The code was extended and built upon an open-source project called Markovify.

You can specify the following settings to generate new words:
  • Corpus - The original text. Words must be separated by spaces or new lines.
  • State Size - Grouping size to determine the next probability
  • Minimum word length - All generated words should be at least this many letters long. Sometimes it is impossible to generate words that are long enough if the state size is too big.
  • Number of words - The number of words that the program returns back as a result
There are two components needed to generate new words - corpus and state size. The corpus is a list of words that the software will break down into letters and extract statistical information from. The state size is the number of consecutive letters that are grouped together, and from which the next probable letter is determined. The best default value for this is generally 2, but you can play around with the slider to get different results.
Let's take the following corpus as an example, with a state size of 2.
  • arc
  • arcs
  • bar
  • bra
  • cab
  • cabs
  • car
  • carb
  • crab
  • scars
The program must now determine a starting letter. The words in the corpus start either with an a, b, c or s. The probabilities of choosing a starting letter are:
  • a = 2/10 = 20%
  • b = 2/10 = 20%
  • c = 5/10 = 50%
  • s = 1/10 = 10%
Let's assume it picks the letter with the highest probability - c. Now it needs to choose the next letter, which can either be an a or an r. The probability of picking the next letter is:
  • c
  • a = 4/5 = 80%
  • r = 1/5 = 20%
Let's assume again that it picks the letter with the highest probability - a. So far, the program has build the word ca. The length of the word is two letters long, which matches our state size. Therefore, from now on, all probabilities need to include the previous 2 letters. The words in the corpus that match our current state are cab, cabs, car, carb and scars. The probability of picking the next letter is:
  • ca
  • b = 2/5 = 40%
  • r = 3/5 = 60%
Again, we assume it picks the letter with the highest probability - r. The word so far is car, and since our state size is 2, the program picks the last 2 letters to determine the probability for the next round. This means that our current state is ar, and the words that match that are arc, arcs, bar, car, carb, scars. The probabilities of the next letter are:
  • car
  • End of word = 2/6 = 33.33%
  • c = 2/6 = 33.33%
  • b = 1/6 = 16.67%
  • s = 1/6 = 16.67%
The program picks c. Our word is now carc, and our current state is rc. The words that match that are arc, arcs. The probabilities of the next letter are:
  • carc
  • End of word = 1/2 = 50%
  • s = 1/2 = 50%
The program picks s. Our word is now carcs, and our current state is cs. The only word in the corpus that matches that is arcs. The probability of the next letter is:
  • carcs
  • End of word = 1/1 = 100%
At this point, the program chooses the End of word token, and returns the generated word - carcs.
